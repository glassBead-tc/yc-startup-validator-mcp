# Comprehensive Q&A Resolution Orchestration

This workflow answers complex questions requiring multi-step reasoning by performing focused searches across Wikipedia, research papers, and expert discussions to synthesize well-rounded, nuanced answers.

## Overview

Complex questions often require understanding multiple facets, synthesizing diverse perspectives, and connecting knowledge from various domains. This orchestration implements a systematic approach to decompose questions, gather relevant information from authoritative sources, and construct comprehensive answers that address all dimensions of the inquiry.

## Required Tools

- `wikipedia_search_exa` - Foundational knowledge
- `research_paper_search_exa` - Academic perspectives
- `web_search_exa` - Current information
- `reddit_search_exa` - Expert discussions
- `youtube_search_exa` - Explanatory content

## Workflow Steps

```
1. Question Decomposition

   - Core question identification
   - Sub-questions extraction
   - Implicit assumptions
   - Context requirements
   - Domain identification
   - Scope boundaries
   - Answer type needed
   - Success criteria

2. Knowledge Gap Mapping
   Identify information needs:
   - Foundational concepts
   - Technical details
   - Current state

   - Controversies

   - Practical applications

3. Search Strategy Planning
   Design approach:

   - Source priorities
   - Query formulations
   - Depth requirements
   - Time allocation
   - Quality thresholds
   - Integration plan
   - Verification needs
```

## Best Practices

1. **Multi-Faceted Approach**: Complex questions need diverse perspectives
2. **Source Hierarchy**: Academic > Expert > Community > General
3. **Acknowledge Uncertainty**: Be clear about confidence levels
4. **Practical Focus**: Always include real-world applications
5. **Iterative Refinement**: Build understanding progressively

## Advanced Techniques

### Question Decomposition Strategies
- Identify core vs peripheral elements
- Map causal relationships
- Recognize hidden assumptions
- Extract measurable components
- Define success criteria

### Cross-Domain Synthesis
- Find analogies across fields
- Identify universal principles
- Map specialized terminology
- Bridge academic-practical gaps
- Create novel connections

### Uncertainty Quantification
- Assign confidence scores
- Track source agreement
- Identify knowledge gaps
- Note temporal validity
- Flag controversial areas

### Answer Optimization
- Layer complexity appropriately
- Provide multiple entry points
- Include visual representations
- Balance depth and clarity
- Ensure actionability

## Common Pitfalls to Avoid

1. **Oversimplification**: Maintain nuance while being clear
2. **Academic Bias**: Include practical perspectives
3. **Recency Bias**: Value established knowledge too
4. **False Certainty**: Acknowledge unknowns
5. **Missing Context**: Provide sufficient background

## Customization Options

### Answer Depth
- **Quick Answer**: Key points only (5 minutes)
- **Standard Answer**: Full analysis (30 minutes)
- **Deep Dive**: Exhaustive treatment (2+ hours)
- **Research Report**: Academic depth (days)

### Audience Adaptation
- **General Public**: Plain language, analogies
- **Students**: Educational focus, examples
- **Professionals**: Industry applications
- **Academics**: Theoretical depth
- **Decision Makers**: Executive summaries

### Question Types
- **How Does X Work?**: Mechanism focus
- **Why Does Y Happen?**: Causal analysis
- **What If Z?**: Scenario exploration
- **Should We A?**: Decision support
- **What's the Future of B?**: Trend analysis
